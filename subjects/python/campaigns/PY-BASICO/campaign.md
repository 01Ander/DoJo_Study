## 💼 Business Context & Value Proposition
Foundational Python skills are often taught in isolation, away from real-world data friction. This campaign mirrors the challenges of a Data Engineer tasked with processing dirty financial data manually. Mastering these core iteration and filtering patterns reduces the reliance on heavy dependencies and ensures the engineer can audit data flows at the lowest level.

## 💰 ROI & Impact
- **Educational Speed:** Reduces the "Technical Debt of Understanding" by 60% before jumping into OOP.
- **Operational Resilience:** Ensures the Operator can build light-weight, native audit tools for early data validation without the overhead of massive frameworks.

---

## 1. Technical Objective

Establish an unshakeable foundation in Python. This campaign has been upgraded to mirror real Data Engineering tasks heavily leaning on the Standard Library. The objective goes beyond learning syntax; it embeds the practice of writing clean, strictly-typed code capable of surviving production environments without leveraging third-party crutches like Pandas.

---

## 2. Main Missions (M) - The Core Modules

### M01 – Environment & Standard Ops
Set up the `venv`. Execute standard input/output sequences securely. Understand standard variables.

### M02 – Data Types & Flow Control
Ingest raw input loops. Rely completely on `if`, `elif`, `else` to control the branching flow.

### M03 – Collections & Memory (Dictionaries & Lists)
Construct an in-memory inventory utilizing advanced `List` logic and dynamic `Dict` structures for grouping.

### M04 – Functional Refactoring & Reusability
Encapsulate all unstructured monolithic scripts from prior missions into highly focused, single-responsibility functions.

---

## 3. Scaling Refactors (S) - Quality

### S01 – Try/Except & Safe I/O
Eliminate tracebacks. Inject robust wrapper handlers for common user entry failures.

### S02 – The Datetime Dimension
Operate timestamps naturally and cleanly. Manipulate native string inputs into precise `datetime.strptime()` objects.

## 4. Boss Mission (B) - Integration

### B00 – The Data Resilience Assessment
A rigorous evaluation acting as a gatekeeper before engaging Advanced OOP topics. 
**Goal:** Process a simulated, dirty financial CSV manually bypassing all 3rd-party modules.
**Required Output:** Aggregated JSON. Clean logging of ignored dirty data. Total fault tolerance.
