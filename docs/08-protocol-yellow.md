# Protocol Yellow (Graceful Degradation)

> *"Protocol Yellow is an Engineering Tool, not a failure. It prioritizes long-term consistency over short-term perfection."*
> — DoJo Philosophy

## Overview
Protocol Yellow is a universal cognitive resilience mechanism designed to ensure the Operator's (Andersson) mental sustainability. It prevents technical and linguistic burnout by enabling a **Graceful Degradation** of the system's demands when the learning friction becomes too high, regardless of the current campaign or mission complexity.

## The Trigger Mechanism
- **Friction Level > 7**
- The Friction Level is self-assessed and declared by the Operator at any point during a session or mission.
- **Immediate Effect:** All strict "Definitions of Done" and "Hard Constraints" are temporarily suspended.

### 🛡️ The Study Gatekeeper (Pre-session Check)
Before starting any *Deep Work* session, the Operator must evaluate their internal state using Protocol Yellow.
- **Decision Rule:** If initial fatigue or cognitive friction is already > 6 before starting, the session should be aborted immediately in favor of a **Rest Day**.
- Do not attempt to "force" a session if the hardware is already in a state of degradation.

## Generalized Mitigation Levers
When activated, the Operator is authorized to pull any or all of the following generic levers to lower cognitive load:

1. **De-abstraction (Scale Down to Familiarity):** 
   Retreat from the target architectural paradigm back to a comfortable baseline.
   * *Example:* If blocked by Object-Oriented patterns, revert to procedural functions. If blocked by complex frameworks (e.g., Docker, Pandas), revert to native execution and standard libraries.
2. **Language Switch (Cognitive Relief):** 
   If processing new engineering concepts *and* English simultaneously causes mental gridlock, the English constraint drops.
   * *Action:* Authorize 100% Spanish for documentation, docstrings, variable names, and mental logic mapping.
3. **Environmental Simplification (Isolate the Logic):** 
   Remove massive real-world complexities to focus strictly on making the algorithm work.
   * *Example:* Bypass authentication steps; replace complex file ingestions (`.csv`, databases, APIs) with hardcoded local variables or mock structures in the same file.
4. **Mission Segmentation (Micro-Goals):** 
   Abandon the overall mission objective. Define a single, tiny micro-goal (e.g., "Just assure the `for` loop prints correctly") and celebrate that as a successful session.
5. **Mandatory Break (The 'Hard Stop'):** 
   If friction persists above level 8 for more than 20 minutes with zero progress, an immediate 15-minute physical separation from the terminal is enforced. If frustration remains, close the session.

## Application by Mission Type
Protocol Yellow adapts its flavor depending on where it is triggered:
- **Assessments / Boss Battles (B-Missions):** If triggered here, it acts as a "Containment Wall". Severe friction on a B-Mission automatically suggests a targeted *Reinforcement Pause* (e.g., a 1-to-2 week detour focusing heavily on building raw muscle memory in the weak area).
- **Core Systems (M-Missions):** It becomes an iterative tool. Build the system "ugly and natively" first, purely to see data flow, then refactor.
- **Side Quests (S-Missions):** Automatically consider aborting or postponing the S-Mission to prioritize Core mental energy.

## The Re-entry Path
Protocol Yellow is a temporary state. To return to the standard Operational Level once the technical foundation feels solid again, execute a progressive overload:

1. **Re-evaluate:** Confirm that the internal Friction Level has dropped to 5 or lower.
2. **Atomic Refactoring:** Take the simplified script built under Protocol Yellow and refactor *just one single component* up to the demanded standard (e.g., converting a single function into a class, or adding types to just one file).
3. **Progressive Language Overload:** Translate the docstrings and comments back into Professional English.
4. **Re-attach the Environment:** Swap out the mock data for real data ingestion, restore the architectural complexity, and execute the final tests.
